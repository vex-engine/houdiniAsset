# 블럭 이동 규칙 (형이 까먹지 않게)

> 슬라이드 에디터의 블럭 이동/선택 규칙. 피그마 스타일 기반.
>
> 작성: 2026-04-24 / 엔진 버전: **v1.1** 확정
>
> 관련 문서: `CHANGELOG.md` (v1.1), `engine/BLOCK_SYSTEM.md` (설계)

---

## 한 줄 요약

**클릭 = 선택. 드래그 = 이동. Alt + 드래그 = 복제. 화살표 = 미세 이동.**

"선택"과 "이동"은 완전히 다른 동작이다. 이걸 헷갈리면 안 된다.

---

## 블럭의 3가지 상태

| 상태 | 외관 | 할 수 있는 일 |
|---|---|---|
| **IDLE** (선택 안 됨) | 아무 표시 없음 | 그냥 보는 상태 |
| **SELECTED** (선택됨) | **진한 녹색** 테두리 | 이동, 삭제, 복제, 감싸기 |
| **EDITING** (편집 중) | **연한 녹색 점선** + 커서 | 글자 입력/수정 |

한 번에 한 블럭만 SELECTED 또는 EDITING. Shift+클릭으로 여러 개 선택 가능.

---

## 선택하는 법 (클릭 규칙)

### 빈 슬라이드 → 블럭 클릭
- **첫 클릭**: 가장 바깥 부모 블럭이 선택됨 (진한 녹색)
  - 예: `조카라스` 카드를 클릭하면 **카드 전체**가 먼저 선택

### 같은 블럭을 다시 클릭
- **한 겹 안으로 파고들기** — 자식 블럭이 선택됨 (여전히 진한 녹색)
  - 예: 카드 → 또 클릭 → `조카라스` 글자 블럭 선택

### 리프(가장 안쪽) 블럭을 또 클릭
- **편집 모드 진입** (연한 녹색 점선)
  - 예: `조카라스` 글자 블럭 → 또 클릭 → 커서 깜빡, 글자 수정 가능

### 더블클릭
- **리프까지 즉시 파고들어 편집 모드**
  - 카드 위에서 더블클릭 = 바로 안쪽 글자 편집
  - 한 번에 편집하고 싶을 때

### 빈 공간 클릭
- **선택 해제** (IDLE로 복귀)

### Shift + 클릭
- **다중 선택 토글**
- 이미 선택된 블럭은 Shift+클릭으로 해제
- 다중 선택 상태에선 편집 모드 진입 불가

### ESC 키
- **한 단계씩 뒤로**
- EDITING → SELECTED (같은 블럭)
- SELECTED → 부모 블럭 SELECTED
- 최상위 SELECTED → IDLE (선택 해제)

## 이동/복제하는 법

### 1. 그냥 드래그 (주력 방법)

- **텍스트/일반 블럭 또는 미디어 본체를 직접 드래그**
- 놓을 때까지 원하는 위치로 이동
- 원래 레이아웃에 있던 텍스트는 `relative + left/top`으로 움직인다.

### 2. Alt + 드래그 (복제)

- 원본은 그대로 두고, 단일 복제본만 생성해 드롭 위치로 이동
- 복제본은 `absolute` 좌표로 배치한다.
- 복제 시 선택/편집/runtime/component/instance 상태를 이어받지 않는다.

### 3. Space + 드래그 (선택 이동)

- 기존 선택 블럭을 이동한다.
- 다중 선택 상태에서는 선택된 블럭들이 함께 이동한다.

### 4. 화살표 키 (미세 이동, "Nudge")

- 블럭 선택된 상태에서:
  - **↑ ↓ ← →** : 1px씩 이동
  - **Shift + ↑↓←→** : 10px씩 이동
- 정밀한 위치 조정용
- 피그마의 Nudge 방식

### 5. 원 위치로 복귀

- 작은 블럭(카드 안쪽 텍스트 등)은 이동하면 부모 레이아웃과 어긋날 수 있음
- 툴바의 **"⟲ 원위치"** 버튼 → `top/left` 리셋 → 부모 레이아웃 기본 위치로
- 단축키: **Ctrl+0 (영)**

---

## 다중 선택 + 이동

- Shift+클릭으로 여러 블럭 선택 (전부 진한 녹색)
- 그 상태에서 Space + 드래그 = **모두 같은 거리만큼 함께 이동**
- 화살표 키도 동일하게 작동 (모두 함께 이동)

---

## 블럭 이동에 대한 핵심 원칙 (기억할 것)

1. **이동과 복제는 다르다**: 그냥 드래그는 이동, Alt+드래그는 복제.
2. **원본 텍스트는 absolute로 바꾸지 않는다**: 레이아웃 흐름이 깨져 전체 텍스트가 밀릴 수 있다.
3. **복제본만 absolute로 둔다**: Alt/Ctrl+D 복제본은 슬라이드 좌표로 고정한다.
4. **핸들은 선택된 블럭에만 붙인다**: 오래된 복제본 핸들이 남아 다른 블럭을 움직이는 일을 막는다.
5. **진한 녹색 = 이동 준비 완료**: 진한 녹색 보이면 드래그 가능.
6. **연한 녹색 = 글자 편집 중**: 이때 이동하려면 먼저 ESC로 선택 모드 복귀.
7. **빈 공간 클릭하면 무조건 선택 해제**: 헷갈리면 빈 곳 한 번 클릭.
8. **최소 단위까지 파고들려면 더블클릭**: 한 번에 안쪽 편집.

## 단축키 전체 목록

### 선택
| 키 | 동작 |
|---|---|
| Click | 한 겹 파고들기 선택 |
| Double-click | 최안쪽까지 즉시 파고들어 편집 |
| Shift+Click | 다중 선택 토글 |
| ESC | 한 단계 바깥으로 |

### 이동
| 키 | 동작 |
|---|---|
| **드래그** | **블럭 이동 (주력)** |
| **Alt + 드래그** | 단일 블럭 복제 후 복제본 이동 |
| **Space + 드래그** | 선택 블럭 이동 |
| ↑↓←→ | 1px 미세 이동 |
| Shift + ↑↓←→ | 10px 미세 이동 |
| Ctrl+0 | 원 위치 복귀 |

### 편집/조작
| 키 | 동작 |
|---|---|
| Enter / F2 | 선택 → 편집 모드 |
| Delete / Backspace | 블럭 삭제 (선택 시) / 글자 삭제 (편집 시) |
| Ctrl/Shift + Delete | 편집 중에도 블럭 통째 삭제 |
| Ctrl+D | 블럭 복제 |
| Ctrl+G | 선택된 블럭(들)을 카드로 감싸기 |
| Ctrl+Shift+G | 감싸진 카드 풀기 |
| Ctrl+Z / Ctrl+Shift+Z | 실행취소 / 재실행 |

## 실전 예시: `조카라스` 카드에서 뭘 하고 싶은가?

### 예시 1: 카드 전체를 옆으로 옮기고 싶다
1. 카드 한 번 클릭 (진한 녹색 — 카드 전체 선택)
2. Space 누른 채로 드래그 → 원하는 위치에

### 예시 2: 카드 안의 `조카라스` 글자만 살짝 위로 올리고 싶다
1. 카드 클릭 (카드 선택)
2. `조카라스` 글자 다시 클릭 (자식 선택, 여전히 진한 녹색)
3. Shift+↑ (10px 위로)

### 예시 3: 글자 내용 수정하고 싶다
1. 카드 더블클릭 → 바로 편집 모드 (연한 녹색 점선 + 커서)
2. 글자 수정

### 예시 4: 카드 두 개를 동시에 이동
1. 첫 카드 클릭
2. 둘째 카드 Shift+클릭 (둘 다 진한 녹색)
3. Space + 드래그 → 둘 다 같이 이동

### 예시 5: 작은 블럭 위치 망쳤을 때
- 블럭 선택 상태에서 Ctrl+0 → 원 위치 복귀

### 예시 6: 실수로 블럭 선택한 거 해제하고 싶다
- 빈 공간 클릭, 또는 ESC 반복

## 왜 Space 키로 이동하게 했나?

- **피그마/스케치/어도비 XD의 표준** — 디자인 툴 쓰는 사람이면 손에 익은 방식
- **선택과 이동의 완벽한 분리** — 실수로 블럭 밀리는 사고 없음
- **드래그 핸들(⠿ 점) 제거 가능** — UI 깨끗
- 피그마에선 실제로는 Space = 캔버스 pan이지만, 우리 에디터에선 캔버스가 고정이라 Space 역할을 블럭 이동으로 전용
