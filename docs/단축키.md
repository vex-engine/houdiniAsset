# 단축키 레퍼런스

최종 업데이트: 2026-04-24 (v2.1 — 스텝 통합 정책)

> **정책 원칙 (2026-04-24 이후)**
> 
> 프레젠테이션 모드에서 **"스텝 단위(블록 하나씩)" 와 "슬라이드 단위(페이지 통째)"** 을 입력 수단이 아니라 **의도** 로 구분한다.
> - 스텝 단위: **휠 · 스페이스 · 방향키 (← → ↑ ↓)**
> - 슬라이드 단위: **Page Up · Page Down**
> - 점프: Home · End
>
> 즉 화살표는 더 이상 "페이지 통째 넘김" 이 아니다. Space·휠과 같이 블록 단위 진행이다.
> 슬라이드를 통째로 넘기려면 Page Up/Down 을 써야 한다.

---

## 🖥️ 프레젠테이션 모드

### 스텝 단위 진행 (블록 하나씩 나타남)
| 키 / 입력 | 동작 |
|---|---|
| `→` `↓` `←` `↑` | 다음/이전 스텝 (블록 하나씩) |
| `Space` | 다음 스텝 |
| 마우스 휠 아래 | 다음 스텝 |
| 마우스 휠 위 | 이전 스텝 |

스텝이 슬라이드의 마지막에 도달하면 **자동으로 다음 슬라이드로 전환** (진입 시 step=0 — 아무것도 안 보인 상태로 시작).
스텝이 0 인데 뒤로 가면 **이전 슬라이드로 전환** (진입 시 step=max — 모든 블록 보인 상태).

### 슬라이드 단위 이동 (페이지 통째)
| 키 | 동작 |
|---|---|
| `Page Down` | 다음 슬라이드 (step=max 상태) |
| `Page Up` | 이전 슬라이드 |
| `Home` | 첫 슬라이드 |
| `End` | 마지막 슬라이드 |

### 터치 (모바일)
| 동작 | 결과 |
|---|---|
| 왼쪽 스와이프 | 다음 슬라이드 (Page Down 과 동일) |
| 오른쪽 스와이프 | 이전 슬라이드 |

### 기타
| 키 | 동작 |
|---|---|
| `F` | 전체화면 토글 |
| `N` | 발표자 노트 표시 토글 |
| `E` | 에디터 모드 진입/종료 (원본에서만) |

### 경계 동작
- **첫 슬라이드에서 더 이전으로 가려는 입력**: 무시, 휠 누적값 리셋
- **마지막 슬라이드의 마지막 스텝에서 더 다음으로 가려는 입력**: 무시, 휠 누적값 리셋
- **첫 슬라이드는 항상 전체 보임** (스텝 0 상태가 없음)

## ✏️ 에디터 모드

### 페이지 이동 (슬라이드 단위 즉시 전환 — 블록 애니 스킵)
| 키 / 입력 | 동작 |
|---|---|
| `Page Down` | 다음 슬라이드 |
| `Page Up` | 이전 슬라이드 |
| 마우스 휠 ↓ (캔버스 위) | 다음 슬라이드 |
| 마우스 휠 ↑ (캔버스 위) | 이전 슬라이드 |

> 에디터 모드에서는 블록 애니메이션이 의미 없으므로 모든 페이지 이동이 슬라이드 단위.

### 파일 / 저장
| 키 | 동작 |
|---|---|
| `Ctrl + S` | 원본 파일 저장 |
| `Ctrl + Shift + S` | 프레젠테이션용 파일 익스포트 |

### 편집 히스토리
| 키 | 동작 |
|---|---|
| `Ctrl + Z` | 실행 취소 (텍스트 편집 중엔 브라우저 기본 동작) |
| `Ctrl + Shift + Z` | 다시 실행 |
| `Ctrl + Y` | 다시 실행 |

### 블럭 선택 (Figma 방식)
| 키 / 동작 | 결과 |
|---|---|
| `Click` | 최상위 블럭 선택 / 같은 블럭 재클릭 = 한 겹 파고들기 |
| `Double-click` | 리프까지 즉시 파고들어 편집 모드 진입 |
| `Shift + Click` | 다중 선택 토글 |
| `ESC` | 편집→선택 / 선택→부모 / 최상위→선택 해제 |
| `Enter` / `F2` | SELECTED → EDITING 전환 |

### 블럭 이동
| 키 / 동작 | 결과 |
|---|---|
| **`드래그`** | 텍스트/일반 블럭 이동 |
| **`Alt + 드래그`** | 단일 블럭 복제 후 복제본 이동 |
| **`Space + 드래그`** | 기존 선택 블럭 이동 |
| **미디어(`ed-media-wrap`)는 그냥 드래그로 이동 가능** | Space 없어도 됨 |
| `↑` `↓` `←` `→` | 1px 미세 이동 (Nudge) |
| `Shift + 방향키` | 10px 이동 |
| `Ctrl + 0` | 원 위치 복귀 (top/left 스타일 리셋) |

### 블럭 조작
| 키 | 동작 |
|---|---|
| `Ctrl + D` | 선택 블럭 복제 |
| `Delete` / `Backspace` | SELECTED면 블럭 삭제 / EDITING이면 글자 삭제 |
| `Ctrl/Shift + Delete` | EDITING 중에도 블럭 통째 삭제 |
| `Ctrl + G` | 선택 블럭(들)을 카드로 감싸기 (= `W` 별칭, 피그마 호환) |
| `Ctrl + Shift + G` | 감싸진 카드 풀기 |
| `W` | **(v2.1)** Wrap in Card — 카드(시각 스타일) + 자식 순차 등장 자동 ON |
| `Shift + W` | **(v2.1)** Wrap in Frame — 투명 컨테이너 (스타일 없음) |
| `Ctrl + Alt + K` | **(v2.1)** Create Component (선택 블럭을 마스터로 등록) |
| `Ctrl + Alt + B` | **(v2.1)** Detach Instance (인스턴스 → 자유 카드) |

### 표시 옵션
| 키 | 동작 |
|---|---|
| `G` | 그리드 표시 토글 |
| `E` | 에디터 모드 진입/종료 |

## 💡 요약

- **스텝 단위 (블록 하나씩)**: 휠 · Space · 방향키 ← → ↑ ↓
- **슬라이드 단위 (페이지 통째)**: Page Up · Page Down
- **점프**: Home · End

에디터 모드

## 🔒 안전장치

에디터 모드의 휠 네비게이션은 다음 상황에서 비활성:
- `contenteditable` 텍스트 편집 중
- 블록 드래그/리사이즈 중 (`.ed-dragging`, `.ed-resizing`)
- `<input>` / `<textarea>` 포커스 중
- 사이드바 영역 (슬라이드 프레임 밖)

## 🧭 검증 도구

단축키가 이상하게 동작하면 먼저:
1. `F12` 콘솔 — 빨간 경고 배너 (`⚠ 엔진 로드 실패`) 떠있는지 확인
2. `bash scripts/verify_html.sh` — HTML 파일 무결성 확인
3. `Ctrl+F5` 하드 리프레시 (캐시가 구버전을 물고 있을 수 있음)

## 📍 코드 위치 (유지보수용)

- 키 매핑: `engine/presentation.js` 의 `document.addEventListener('keydown', …)` 블록
- 휠 처리: 같은 파일 `CF.addEventListener('wheel', …)` 블록
- 스텝 로직: `advanceStep(dir)` 함수 — 휠/스페이스/화살표 공통 진입점
- 슬라이드 전환: `go(i,dir)` (슬라이드 단위) / `goStep(i,dir,entryStep)` (스텝 진입 제어)
- 이 문서를 수정하면 `engine/presentation.js` 의 키 매핑 주석과 동기화할 것.
