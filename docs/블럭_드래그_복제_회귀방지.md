# 블럭 드래그/복제 회귀 방지 메모

작성: 2026-04-28

## 결론

- 일반 드래그: 잡은 블럭만 이동한다.
- Alt + 드래그: 원본은 건드리지 않고 복제본만 만든 뒤 이동한다.
- Ctrl+D: Alt 복제와 같은 복제 헬퍼를 사용한다.

## 재발 방지 규칙

1. 원본 텍스트 블럭을 이동할 때 `position:absolute`로 바꾸지 않는다.
   - 원본 텍스트는 슬라이드의 기존 flex/layout 흐름에 남아 있어야 한다.
   - 이동은 `position:relative` 상태에서 `left/top`만 조정한다.

2. 복제본만 `position:absolute`로 만든다.
   - Alt + 드래그와 Ctrl+D 복제본은 슬라이드 좌표로 배치한다.
   - 복제본은 DOM 흐름에 끼워 넣지 말고 slide에 append한다.

3. 전체 슬라이드 블럭을 이동/복제 중에 좌표 고정하지 않는다.
   - `freezeSlideBlocksToAbsolute(slide)` 같은 전체 변환은 레이아웃 텍스트를 흐트러뜨린다.
   - 필요한 경우에도 잡은 블럭 하나만 변환한다.

4. 핸들은 선택된 블럭에만 붙인다.
   - 모든 블럭에 `.ed-block-resize*` 핸들을 붙이면 오래된 복제본/선택 잔재가 같이 움직이는 버그가 재발한다.

5. 복제는 반드시 `cloneBlockForDuplicate()`를 거친다.
   - 제거 대상: 선택/편집/드래그/리사이즈 클래스, `contenteditable`, component/instance/runtime 속성, editor artifact.

## 공통 구현 위치

- `engine/editor/editor.block.js`

  - `attachHandles()`

## 테스트 체크리스트

- Alt + 드래그를 10회 이상 반복해도 원본 텍스트 위치가 변하지 않아야 한다.
- 복제본을 여러 번 이동해도 제목/부제/코멘트가 아래로 밀리지 않아야 한다.
- 스케일 핸들을 잡아도 선택한 블럭 외 텍스트가 재배치되지 않아야 한다.
- Ctrl+D 복제 후 이동해도 Alt 복제와 같은 동작이어야 한다.
