핵심 설계 원칙 (하드코딩 회피)

State는 DOM에 산다모든 새 기능 상태(data-group-mode, data-layout, data-gap 등)는 DOM data-attribute에만 저장. JS 전역변수 미러 X. undo/redo가 deck 스냅샷이라 자동으로 따라옴.
JS는 attribute, CSS는 시각JS는 setAttribute + setProperty(--ed-gap 등 CSS variable)만 박음. flex-direction, gap, padding 적용은 CSS selector가 처리. 관심사 분리.
BLOCK API 단일 진입점모든 wrap/unwrap/setLayout/makeComponent는 BLOCK 객체와 EA API를 거침. 외부에서 DOM 직접 조작 금지.
백워드 호환기존 groupBlocksWrap / ungroupBlockUnwrap 호출자 변경 없음. 내부에서 wrapBlocks / unwrapBlock으로 위임.
애니메이션 자동 연동Card로 wrap 시 data-anim-children="seq" 자동 부여 → presentation.js의 stepAuto 시스템이 자식들에게 자동 step 부여. wrap한 의미가 즉시 살아남.
삭제 가드Component 마스터 삭제 시 인스턴스 N개도 함께 삭제 confirmDlg. ed-group 삭제 시 자식 트리 전체 삭제 (정상). 핸들 자동 재동기화.
