### SLIDE 03 — 핵심 메시지
# 프롬프트는 명령어가 아니라 콘티다

**AI한테 말하는 법 ≠ 프로그래밍**
**AI한테 말하는 법 = 후배 아티스트한테 콘티 주기**

- 우리는 이미 콘티 짜는 법을 안다
- 라이팅, 카메라, 재질, 분위기 — 매일 쓰는 언어
- 이걸 그대로 옮기면 됨

> **화자 노트:** "프롬프트 엔지니어링이라는 단어 때문에 어렵게 느껴지지만, 본질은 콘티입니다. 우리가 외주 컨셉 작가한테 일 맡길 때 뭐라고 하시나요? '낮 시간대, 와이드샷, 폐허 분위기, 동양적 디테일' — 이게 이미 프롬프트입니다. 새로 배우는 게 아니라 있는 능력을 다른 도구에 쓰는 거예요."

### SLIDE 04 — 워크플로우 전체도
# 오늘의 파이프라인

```
┌──────────┐    ┌──────────────┐    ┌─────────────┐
│  GROK    │ →  │  MIDJOURNEY  │ →  │ NANO BANANA │
│ 컨셉 발산  │    │  스타일 락인   │    │   최종 조립    │
└──────────┘    └──────────────┘    └─────────────┘
   30분            10분                20분
   "탐색"          "톤 결정"            "정밀 수정"
```

| 단계 | 역할 | 산출물 |
|------|------|--------|
| **Grok** | 컨셉 대량 발산 | 방향성 후보 20~30장 |
| **Midjourney** | 톤/스타일 고정 | 스타일 레퍼런스 |
| **Nano Banana** | 정밀 편집/조립 | 최종 레퍼런스 보드 |

> **화자 노트:** "이 한 장이 오늘 강의의 전부입니다. 사진 찍어두세요. 단계별로 다 잘하는 도구가 다릅니다. 하나만 쓰면 절반의 결과만 나와요. 발산은 그록, 락인은 미드저니, 조립은 나노바나나. 각자의 장점만 빼서 쓰는 파이프라인입니다." (AD 출신 팀장에게 어필되는 슬라이드)

### SLIDE 05 — Why Grok First?
# 1단계: 왜 컨셉은 그록인가

**🏗️ 파이프라인 관점**
- 검열이 약함 → 다크 판타지/폐허/전투씬에 막힘 없음
- 대화형 반복 → 한 세션에서 30개 변형 누적
- Speed 모드 → 탐색 비용 사실상 0
- **포지션: "발산 단계 전용"**

**⚡ 지금 당장 한 줄**
> 기획서 한글 텍스트 그대로 그록에 던지고:
> *"이걸 영어 이미지 프롬프트 5개 변형으로 만들어줘. 분위기는 each 다르게."*

→ 영어 작성 부담 0, 5개 비교 가능

> **화자 노트:** "주니어분들 영어 프롬프트 쓰는 게 부담이시죠? 그록은 한글로 던져도 알아서 영어 프롬프트를 만들어줍니다. 그리고 한 번에 5개씩 변형을 요청하는 게 핵심이에요. 한 개씩 뽑으면 비교가 안 됩니다."

### SLIDE 06 — Grok 컨셉 발산 실전
# 그록으로 30분 만에 컨셉 30장

**[Grok 발산 데모 영상 재생 — 1분]**

**🏗️ 파이프라인 관점**
- 1라운드: 환경 5개 → 방향 선택
- 2라운드: 라이팅 5개 → 시간대 선택
- 3라운드: 카메라 5개 → 앵글 선택
- 점진적 수렴 (한 번에 하나만 바꿈)

**⚡ 지금 당장 한 줄**
> *"이 중에 3번째 마음에 들어. 같은 구도에서 라이팅만 골든아워, 블루아워, 한밤중 3가지 변형 만들어줘."*

> **화자 노트:** 영상 재생 전에 "여기서 보실 건 — 한 번에 모든 걸 바꾸지 않고, 한 가지 변수만 바꿔가면서 수렴하는 과정입니다." 영상 끝나고 한 줄 카드 강조.

### SLIDE 07 — Midjourney로 스타일 락인
# 2단계: 미드저니에서 스타일을 가져온다

**[MJ → Grok 스타일 적용 영상 재생 — 1분]**

**🏗️ 파이프라인 관점**
- MJ의 강점: 시네마틱 톤/색감의 압도적 완성도
- 단점: 대화형 수정이 약함 → 발산엔 부적합
- **해법: MJ에서 톤만 추출해서 그록으로 가져온다**
- `--sref` (스타일 레퍼런스)로 톤 고정

**⚡ 지금 당장 한 줄**
> MJ에서 마음에 드는 톤 1장 뽑고:
> *"이 이미지 스타일을 그대로 유지하면서, 다른 환경 5개 만들어줘."* (그록에 이미지 첨부)

> **화자 노트:** "이게 진짜 핵심 트릭입니다. 미드저니는 톤이 압도적인데 수정이 답답해요. 그록은 수정은 잘 되는데 톤이 평범해요. 그래서 미드저니에서 톤만 한 장 뽑고, 그걸 그록에 레퍼런스로 넣어버립니다. 두 도구의 장점만 합치는 거예요. 영상 보시죠."

### SLIDE 08 — Nano Banana로 최종 조립
# 3단계: 나노바나나로 정밀 수정

**[Nano Banana vs Grok 비교 이미지 ① 표시]**

**🏗️ 파이프라인 관점**
- 강점: 정확한 수정 지시가 먹힘
- 레퍼런스 이미지 14장까지 입력 가능
- "여기 안개", "이 부분 어둡게" 같은 부분 수정에 강함
- **포지션: "최종 마무리 편집기"**

**⚡ 지금 당장 한 줄**
> *"이 이미지 그대로 두고, 왼쪽 기둥 옆에 횃불 추가해줘. 빛은 따뜻한 주황색."*

→ 부분만 정확히 바뀜

> **화자 노트:** "그록에서 컨셉 잡고, 미드저니에서 톤 잡았으면, 마지막은 나노바나나에서 디테일 수정입니다. 나노바나나의 킬러 기능은 부분 수정이에요. 그록에 '여기 횃불 넣어줘'라고 하면 이미지 전체가 바뀌어버리는데, 나노바나나는 그 부분만 정확히 바꿔줍니다."
