> 구조

핵심
아키텍처

// fixed + absolute + transition

핵심 아키텍처를 한 문장으로 요약.

4가지 핵심

  • 뷰포트 풀스크린 — position: fixed + inset: 0
  • 슬라이드 겹침 — position: absolute, Flexbox 정렬
  • CSS Transition — GPU 가속 (transform + opacity)
  • 빌드 효과 — data-step 순차 등장
4가지 핵심 구조를 하나씩 설명.

레이아웃

풀스크린

position: fixed로 뷰포트 전체를 차지. 여백 없이 모든 공간 활용.

16:9 고정

min() 함수로 비율 고정. 비율 불일치 시 레터박스 발생.

이 프레젠테이션은 풀스크린 방식

풀스크린과 16:9 방식의 차이를 비교.

전환 코드

engine.js 
function goToSlide(index, direction) {
  if (state.isTransitioning) return;
  state.isTransitioning = true;

  const current = slides[state.currentSlide];
  const next = slides[index];

  next.classList.add('active');
  current.classList.remove('active');
}

// isTransitioning 플래그로 연속 입력 방지

하이라이트된 줄이 핵심. isTransitioning 잠금 패턴.

> 조작

네비게이션

키보드 / 터치 / URL Hash / 프로그레스 바

실제 키보드와 스와이프를 시연.

입력 시스템

  • 키보드 — ← → Space PageUp/Down Home End
  • URL Hash — pushState + popstate 뒤로가기
  • 터치 스와이프 — 임계값 50px, 300ms, 30도
  • 프로그레스 바 — CSS width transition
키보드, 터치, URL 동기화를 각각 설명.

빌드 효과

style.css 
/* 숨김 상태 */
[data-step] {
  opacity: 0;
  transform: translateY(20px);
  transition: all 0.5s ease;
}

/* 표시 상태 */
[data-step].visible {
  opacity: 1;
  transform: translate(0, 0);
}

// opacity + transform = GPU 가속 보장

CSS만으로 빌드 효과를 구현하는 핵심 코드.
